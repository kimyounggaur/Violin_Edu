# 🎻 Project: SJ Artiee - Violin Fingering Master Web App
**Role**: 당신은 세계 최고의 프론트엔드 개발자이자, 에듀테크(EdTech) UI/UX 디자이너입니다. 사용자가 바이올린 운지법(Fingering)과 계이름을 가장 직관적으로 학습할 수 있는 웹앱을 개발해야 합니다.

## 1. Tech Stack & Design Philosophy
* **Framework**: Next.js (App Router), React, TypeScript
* **Styling**: Tailwind CSS
* **Animation**: Framer Motion (부드럽고 직관적인 인터랙션)
* **Audio**: 브라우저 기본 Web Audio API (또는 Tone.js)
* **Design System (Core Vibe)**: 
    * 노션(Notion) 스타일의 미니멀하고 깨끗한 UI/UX.
    * 화이트(#FFFFFF)와 오프화이트(#F9F9F9) 배경, 텍스트는 짙은 차콜(#2D2D2D).
    * 불필요한 장식을 배제하고, 바이올린 지판(Fingerboard)과 계이름 자체에 시선이 집중되도록 설계.
    * 버튼과 인터랙션 요소는 미세한 그림자(sm)와 둥근 모서리(rounded-lg) 적용.

## 2. Step-by-Step Implementation Guide

개발은 다음 5단계로 진행하며, 각 단계가 완벽히 동작하는지 확인한 후 다음 단계로 넘어가세요.

### Step 1: Base Layout & UI Shell 구축
1. 반응형(Mobile-first) 레이아웃을 설정하세요. 화면 중앙에 메인 콘텐츠가 위치하도록 합니다.
2. 상단 헤더에 "SJ Artiee: 바이올린 운지법 마스터" 타이틀을 깔끔한 Sans-serif 폰트로 배치하세요.
3. 메인 화면을 두 영역으로 나눕니다: 
   - 상단: 선택된 음의 정보(계이름, 영어 음이름, 오선보 위치)를 크게 보여주는 디스플레이 영역.
   - 하단: 인터랙티브 바이올린 지판(Fingerboard) 영역.

### Step 2: Interactive Fingerboard (지판) 컴포넌트 개발
1. 4개의 현(G, D, A, E)을 가로(또는 세로) 형태로 시각화하세요. 현의 굵기는 G현이 가장 두껍고 E현이 가장 얇게 표현합니다.
2. 제1포지션(1st Position)을 기준으로, 개방현(0)부터 4번 손가락(4)까지 누를 수 있는 위치(테이프 위치)를 점선이나 둥근 마커로 은은하게 표시하세요.
3. 사용자가 특정 마커를 클릭(또는 터치)하면:
   - 해당 마커가 활성화된 색상(예: 노션의 파란색 #2EAADC)으로 부드럽게 채워짐 (Framer Motion 사용).
   - 진동하듯 짧고 부드러운 애니메이션 효과 적용.

### Step 3: Music Theory Data & Mapping (음악 이론 매핑)
다음 데이터를 상태(State)로 관리하고 지판과 연동하세요.
* **G현 (4번줄)**: 솔(G3), 라(A3), 시(B3), 도(C4), 레(D4)
* **D현 (3번줄)**: 레(D4), 미(E4), 파#(F#4), 솔(G4), 라(A4)
* **A현 (2번줄)**: 라(A4), 시(B4), 도#(C#5), 레(D5), 미(E5)
* **E현 (1번줄)**: 미(E5), 파#(F#5), 솔(G5), 라(A5), 시(B5)
* *조건*: 사용자가 지판을 누르면, 상단 디스플레이 영역에 "D현 1번 손가락 - 미(E4)" 와 같이 직관적인 텍스트가 즉시 업데이트되어야 합니다.

### Step 4: Sound Feedback (청각적 피드백 통합)
1. 사용자가 지판의 마커를 클릭할 때마다 해당 음높이(Pitch)에 맞는 짧고 명료한 바이올린 소리(또는 사인파 합성음)가 재생되도록 Web Audio API를 연동하세요.
2. 소리는 지연 없이(Zero-latency) 즉각적으로 재생되어야 합니다. (시각적 피드백과 청각적 피드백의 완벽한 동기화)

### Step 5: Learning & Quiz Mode (학습 및 퀴즈 모드 도입)
1. 화면 상단에 [탐색 모드]와 [퀴즈 모드]를 전환할 수 있는 토글스위치를 만드세요.
2. **탐색 모드**: 자유롭게 지판을 눌러보며 소리와 계이름을 익히는 기본 모드.
3. **퀴즈 모드**: 
   - 시스템이 무작위로 계이름(예: "A현의 '도#'을 찾아보세요!")을 제시합니다.
   - 사용자가 올바른 지판 위치를 클릭하면 '정답' 시각 효과(초록색 링 웨이브 애니메이션)를 보여줍니다.
   - 틀리면 '오답' 시각 효과(가벼운 흔들림 애니메이션, 빨간색)와 함께 다시 시도하게 합니다.

## 3. Final Polish & Edge Cases
* 모바일 환경에서 터치 영역이 충분히 넓어 오작동이 없도록 마커의 패딩(Padding)을 넉넉하게 설정하세요.
* 컴포넌트를 최대한 모듈화하여, 추후 '첼로'나 '플루트' 등 다른 악기로 확장할 수 있도록 확장성(Scalability) 있는 아키텍처를 유지하세요.
