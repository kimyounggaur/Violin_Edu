# 🎻 바이브코딩 프롬프트 — 바이올린 운지법 & 계이름 학습 웹앱

> 이 문서는 **AI 코딩 도구(Claude, Cursor, Bolt.new, Lovable, v0, Windsurf, Replit Agent 등)** 에 그대로 붙여넣어
> "바이올린 운지법과 계이름을 가장 직관적으로 배우는 웹앱"을 만들기 위한 **단계별 프롬프트 모음**입니다.
> 핵심은 ① 정확한 바이올린 운지 데이터 ② 직관적 UI/UX 명세 ③ 한 단계씩 검증하며 쌓는 빌드 순서입니다.

---

## 📌 이 문서 사용법

1. **먼저 [4. 핵심 도메인 데이터]와 [5. 화면 구성]을 한 번 읽어** 무엇을 만드는지 파악합니다.
2. AI 도구에 **[부록 A. 통합 프롬프트]를 먼저 붙여넣어** 전체 맥락을 심어 줍니다.
3. 그다음 **[6. 단계별 빌드 프롬프트]의 STEP 1 → STEP 8을 순서대로** 하나씩 붙여넣고, 각 단계 결과를 확인한 뒤 다음으로 넘어갑니다.
4. 한 번에 다 만들고 싶으면 [부록 A]만 쓰되, 결과 품질은 단계별 진행이 더 좋습니다.

> ⚠️ **중요**: 아래 "운지 데이터"는 음악적으로 검증된 값입니다. AI가 임의로 음높이·손가락 위치를 추측하면 틀리기 쉬우니, **데이터는 반드시 그대로 사용**하라고 명시하세요.

## 🎯 1. 프로젝트 한 줄 정의 & 목표

- **한 줄 정의**: 바이올린 입문자가 *지판 위 손가락 위치(운지법)* 와 *그 음의 계이름(도레미)* 을 보고·듣고·따라 하며 익히는 인터랙티브 웹앱.
- **핵심 목표**
  1. 악보의 음표 ↔ 지판의 손가락 위치 ↔ 실제 소리를 **하나의 화면에서 즉시 연결**한다.
  2. 클릭/탭 한 번으로 **계이름 + 음이름 + 손가락 번호 + 실제 음**을 동시에 보고 들을 수 있다.
  3. "개방현 → 손가락 추가 → 음계 → 동요" 순서로 **막힘없이 단계 학습**한다.
- **차별점**: 텍스트 설명이 아니라 **시각(지판 그림) + 청각(실제 음) + 게임형 반복(퀴즈)** 의 결합.

## 👤 2. 타깃 사용자

- 바이올린을 처음 잡은 성인/청소년/학부모(자녀 지도용).
- 악보를 거의 못 읽고, "이 음표는 어느 줄, 몇 번 손가락이지?"가 가장 큰 장벽인 사람.
- 주로 **태블릿(거치형)·데스크톱**, 일부 모바일에서 사용. → **반응형 필수**, 탭 타깃은 크게.

## 🧰 3. 추천 기술 스택

| 영역 | 선택 | 비고 |
|---|---|---|
| 프레임워크 | **React + Vite** | 단일 페이지, 빠른 핫리로드 |
| 스타일 | **Tailwind CSS** | 빠른 반응형 + 디자인 토큰 |
| 오디오 | **Tone.js** | 음 재생/타이밍. 음이름(`"A4"`, `"F#4"`) 직접 지원 |
| 악보 렌더링 | **VexFlow** (또는 abcjs) | 오선보·음표 표시. MVP에서는 선택 |
| 아이콘 | **lucide-react** | 가볍고 일관됨 |
| 상태관리 | React 내장 `useState`/`useReducer` | 외부 라이브러리 불필요 |

> 브라우저 저장소(localStorage 등)는 환경에 따라 막힐 수 있으니, 진행도는 **메모리 상태(useState)** 로만 관리하고, 필요 시 "저장 기능은 추후"로 둡니다.

## 🎼 4. 핵심 도메인 데이터 (반드시 그대로 사용)

### 4.1 4현 & 1포지션(제1포지션) 운지표

바이올린 4개 줄(낮은음→높은음): **G현 · D현 · A현 · E현**.
초보자 표준인 **제1포지션 기본 손가락 형태**(2번·3번 손가락이 붙는 형태)를 기준으로 합니다.

| 줄(현) | 0 (개방현) | 1번 손가락 | 2번 손가락 | 3번 손가락 | 4번 손가락 |
|---|---|---|---|---|---|
| **G현** (4번 줄) | G · 솔 | A · 라 | B · 시 | C · 도 | (D · 레) |
| **D현** (3번 줄) | D · 레 | E · 미 | F♯ · 파♯ | G · 솔 | (A · 라) |
| **A현** (2번 줄) | A · 라 | B · 시 | C♯ · 도♯ | D · 레 | (E · 미) |
| **E현** (1번 줄) | E · 미 | F♯ · 파♯ | G♯ · 솔♯ | A · 라 | (B · 시) |

**손가락 간격(핵심 시각 포인트)** — 모든 줄에서 동일:

```
개방현(0) —온음— 1번 —온음— 2번 —반음— 3번 —온음— 4번
                              └── 2번·3번 손가락이 서로 붙음(반음) ──┘
```

- 💡 **꿀팁(앱에 표시)**: 각 줄의 **4번 손가락 음 = 바로 위 줄의 개방현 음**입니다.
  (예: G현 4번 = D = D현 개방현) → 음정(인토네이션) 확인용으로 좋습니다.
- 본 데이터는 D장조/A장조 계열에서 자연스럽게 떨어지는 "높은 2번" 형태입니다.
  내림표(♭)·낮은 손가락 변형은 [9. 확장 기능]에서 다룹니다.

### 4.2 손가락 색상 규칙 (앱 전체에서 일관 사용)

| 손가락 | 색상 HEX | 의미 |
|---|---|---|
| 0 (개방현) | `#64748B` (슬레이트) | 손가락 안 씀 |
| 1번 | `#2563EB` (파랑) | 검지 |
| 2번 | `#16A34A` (초록) | 중지 |
| 3번 | `#EA580C` (주황) | 약지 |
| 4번 | `#9333EA` (보라) | 소지 |

> ♿ 색맹 고려: **색 + 숫자**를 항상 함께 표시(색만으로 구분하지 않음).

## 🖥️ 5. 화면 구성 (MVP 전체 그림)

```
┌───────────────────────────────────────────────┐
│  헤더: 로고 🎻 | 모드 탭 [지판 학습][악보 연결][퀴즈][연습곡] │
├───────────────────────────────────────────────┤
│  표시 토글:  ◉계이름  ○음이름  ○손가락번호   🔊소리켜기  │
├───────────────────────────────────────────────┤
│                                               │
│      [ 인터랙티브 지판(SVG) — 화면의 주인공 ]      │
│   G   D   A   E                                │
│   ●   ●   ●   ●   ← 개방현(0)                   │
│   ○   ○   ○   ○   ← 1번                        │
│   ○   ○   ○   ○   ← 2번 ┐ 2·3 붙음             │
│   ○   ○   ○   ○   ← 3번 ┘                      │
│   ○   ○   ○   ○   ← 4번                        │
│                                               │
├───────────────────────────────────────────────┤
│  선택된 음 패널:  "라 (A) · A현 · 1번 손가락"  ▶재생  │
│  (악보 연결 모드일 때 여기 오선보 + 음표가 함께 표시)   │
└───────────────────────────────────────────────┘
```

- **지판이 화면의 60% 이상**을 차지하는 주인공. 나머지는 보조.
- 위가 너트(낮은 손가락), 아래로 갈수록 높은 손가락 — 또는 가로형(왼쪽 너트→오른쪽). **세로형 권장**(태블릿 거치 시 자연스러움).
- 모든 음 위치는 **큰 원형 버튼(탭 타깃 ≥ 44px)**.

## 🪜 6. 단계별 빌드 프롬프트

> 각 STEP의 **회색 박스 전체를 그대로 복사**해 AI에 붙여넣으세요. STEP 1 전에 [부록 A]를 먼저 넣는 것을 권장합니다.

### STEP 1 — 프로젝트 뼈대 + 지판 시각화(정적)

```
React + Vite + Tailwind CSS 프로젝트를 세팅하고, 바이올린 "제1포지션 지판"을
세로형 SVG로 그려줘.

요구사항:
- 4개의 줄(왼쪽부터 G, D, A, E)을 세로 선으로 그린다. 줄 위에 줄 이름 라벨 표시.
- 지판 배경은 흑단색(#2A2118), 줄은 가는 밝은 선.
- 각 줄마다 5개의 음 위치(개방현 0 + 1,2,3,4번)를 위→아래 순서로 원형 버튼으로 배치한다.
- 2번과 3번 위치는 "반음 간격"이라 서로 가깝게(붙도록) 배치하고,
  나머지는 "온음 간격"으로 더 넓게 배치한다.
- 각 원의 색은 손가락 번호별 색상 규칙(0 #64748B, 1 #2563EB, 2 #16A34A, 3 #EA580C, 4 #9333EA)을 따른다.
- 원 안에는 손가락 번호를 표시한다.
- 데이터는 내가 준 FINGERBOARD 객체를 그대로 src/data.js 로 만들어 사용한다. 음높이/손가락을 임의로 바꾸지 말 것.
- 반응형으로 모바일/태블릿/데스크톱에서 깨지지 않게.

지금은 클릭 동작 없이 "정적 시각화"만 완성해줘.
```

**확인 포인트**: 4현이 보이고, 2·3번이 붙어 있고, 색·번호가 맞는지.

---

### STEP 2 — 클릭 시 정보 + 소리 (인터랙션)

```
이제 지판을 인터랙티브하게 만들어줘.

- Tone.js를 설치하고, 사용자가 화면을 처음 탭할 때 AudioContext를 시작한다(브라우저 정책 대응).
- 음 위치 원을 클릭/탭하면:
  1) 해당 음을 강조(테두리 굵게 + 살짝 확대 애니메이션)한다.
  2) 화면 하단 "선택된 음 패널"에 계이름·음이름·줄·손가락 번호를 크게 표시한다.
     예: "라 (A) · A현 · 1번 손가락".
  3) 그 음의 실제 소리를 Tone.js로 재생한다. 음높이는 데이터의 note 값(예 "F#4")을 그대로 사용.
- 바이올린 음색에 가깝도록 Tone.Synth 대신 부드러운 신스(예: Tone.AMSynth 또는 FMSynth)에
  적당한 어택/릴리즈 엔벨로프를 준다. (샘플 기반 Tone.Sampler가 가능하면 더 좋지만, 외부 샘플 URL이
  확실하지 않으면 신스로 근사한다.)
- 헤더에 "🔊 소리 켜기/끄기" 토글을 둔다.
- 마지막으로 누른 음은 계속 강조 상태를 유지한다.
```

**확인 포인트**: 탭하면 강조+정보+소리가 동시에 나오는지, 음높이가 표와 일치하는지.

---

### STEP 3 — 표시 모드 토글 (계이름 / 음이름 / 손가락 번호)

```
원 안에 표시하는 내용을 전환하는 토글을 헤더 아래에 추가해줘.

- 3개 옵션: [계이름(도레미)] [음이름(CDEFG)] [손가락 번호] — 라디오 버튼 형태, 기본값은 "계이름".
- 선택에 따라 모든 원 안의 텍스트가 즉시 바뀐다.
  · 계이름: 솔, 라, 시, 도, 레, 파♯ 등
  · 음이름: G, A, B, C, F♯ 등
  · 손가락 번호: 0,1,2,3,4
- 어떤 모드든 색상(손가락 색)은 그대로 유지한다.
- 추가로 "범례(legend)"를 작은 카드로 보여준다: 색-손가락 매핑 + "2·3번 손가락은 반음(붙음)" 설명.
```

---

### STEP 4 — 악보 연결 모드 (음표 ↔ 지판)

```
"악보 연결" 모드 탭을 추가해줘. (VexFlow 사용, 없으면 SVG로 직접 오선보를 그려도 됨)

- 화면 상단에 높은음자리표(treble clef) 오선보를 그린다. 바이올린은 높은음자리표를 읽는다.
- 두 방향 연동:
  1) 지판에서 음을 누르면 → 오선보에 그 음표가 정확한 위치로 표시되고 계이름이 음표 위에 뜬다.
  2) "다음 음표" 버튼을 누르면 → 오선보에 랜덤(또는 음계 순서) 음표가 뜨고,
     지판에서 그 음 위치가 은은하게 깜빡이며 힌트를 준다.
- 음표를 누르면 해당 음을 재생한다.
- 데이터의 note(예 "C#5")를 오선보 위치 계산에 사용한다. 임시표(♯)도 정확히 표기.
```

**확인 포인트**: 오선보의 음 높이가 실제 음과 맞는지(특히 ♯음).

---

### STEP 5 — 단계별 학습 경로 (레슨)

```
"지판 학습" 모드 안에 단계별 레슨 진행을 넣어줘.

레슨 순서(잠금 해제식):
  1) 개방현 4개(G·D·A·E)만 — 줄 이름과 소리 익히기
  2) D현 + A현의 1·2·3번 손가락
  3) G현·E현 추가 (4현 전체)
  4) 4번 손가락 + "4번=위 줄 개방현" 개념
  5) D장조 음계 한 옥타브 (레미파♯솔라시도♯레)

- 현재 레슨에서 다루는 음만 지판에서 밝게(활성), 나머지는 흐리게(비활성) 표시.
- 각 레슨 상단에 1~2문장 안내 카드.
- "이 레슨 음 전부 한 번씩 눌러보기"를 완료하면 다음 레슨 잠금 해제 + 진행바 표시.
- 진행도는 메모리 상태로만 관리(브라우저 저장소 사용 금지).
```

### STEP 6 — 퀴즈/연습 모드 (게임형 반복)

```
"퀴즈" 모드 탭을 추가해줘. 두 가지 문제 유형을 번갈아 낸다.

유형 A (듣고 찾기): 음을 소리로 들려주고(또는 오선보로 보여주고), 지판에서 맞는 위치를 누르게 한다.
유형 B (이름 맞히기): 지판에서 한 위치를 강조하고, 보기 4개 중 계이름을 고르게 한다.

- 정답: 초록 강조 + 짧은 효과음, 오답: 빨강 흔들림 + 정답 위치 표시.
- 10문제 한 세트, 점수와 정답률을 마지막에 보여주고 "다시 풀기" 버튼.
- 난이도 선택: [개방현+1번] / [1~3번] / [4현 전체].
- 너무 어렵지 않게, 첫 문제는 항상 쉬운 음으로 시작.
```

### STEP 7 — 연습곡 따라치기 모드

```
"연습곡" 모드 탭을 추가해줘. 내가 준 SONGS 데이터를 사용한다.

- 곡 선택 드롭다운(기본: 작은별).
- 곡의 음표 줄을 가사와 함께 가로로 나열하고, 각 음표에 줄+손가락 번호를 색상 배지로 표시.
- "재생" 버튼: tempo에 맞춰 음을 차례로 재생하면서 현재 음표를 하이라이트하고,
  지판에서도 해당 위치를 동시에 강조한다(따라 짚기 가능).
- 속도 조절 슬라이더(50%~120%)와 "한 음씩 넘기기" 버튼 제공.
- 곡 안의 모든 음은 반드시 FINGERBOARD 운지표 범위 안에 있어야 한다.
```
