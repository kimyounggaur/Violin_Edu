# 바이올린 운지법·계이름 학습 웹앱 바이브코딩 프롬프트 설계서

> 목적: 바이올린 초보자가 **운지 위치 → 손가락 번호 → 음이름 → 계이름 → 소리 → 간단한 연주**를 한 화면에서 직관적으로 연결해 배울 수 있는 웹앱을 개발하기 위한, 단계별 바이브코딩용 프롬프트 모음입니다.  
> 사용법: 아래의 **마스터 프롬프트**를 먼저 코딩 AI에게 전달한 뒤, `1단계`부터 순서대로 붙여 넣으며 개발합니다. 각 단계가 끝나면 체크리스트를 통과한 뒤 다음 단계로 진행합니다.

---

## 0. 개발 목표 요약

### 만들 앱의 한 문장 정의

**“바이올린 지판을 탭하면 해당 현, 손가락 번호, 음이름, 계이름, 소리가 즉시 연결되고, 사용자는 짧은 미션을 통해 운지법과 계이름 연주법을 자연스럽게 익히는 반응형 웹앱.”**

### 핵심 사용자

- 바이올린을 처음 배우는 어린이, 청소년, 성인 초보자
- 악보의 계이름은 알지만 바이올린에서 어디를 눌러야 하는지 모르는 학습자
- 개방현, 1포지션, 낮은/높은 손가락 위치를 헷갈리는 학습자
- 교사가 수업 중 태블릿이나 노트북으로 보여줄 수 있는 보조 교구가 필요한 경우

### 최우선 UX 원칙

1. **한 번 탭하면 바로 이해**되어야 한다.
2. 화면에는 항상 `현 이름`, `손가락 번호`, `음이름`, `계이름`, `소리 듣기`가 함께 보여야 한다.
3. 학습자는 “이 음이 어디인지”뿐 아니라 “어떤 손가락으로 누르고 어떻게 연주하는지”까지 알 수 있어야 한다.
4. 바이올린에는 프렛이 없으므로 모든 위치 표시는 “정확한 프렛”이 아니라 **초보자용 위치 가이드**로 설명한다.
5. 처음에는 복잡한 이론보다 `G-D-A-E 개방현`, `1포지션`, `도레미 연주`, `짧은 멜로디`를 중심으로 설계한다.

## 1. 마스터 프롬프트

아래 프롬프트를 코딩 AI에게 가장 먼저 전달하세요.

```text
너는 음악교육 UX에 강한 시니어 프론트엔드 엔지니어이자 바이올린 초급 교육 콘텐츠 설계자다.

목표는 “바이올린 운지법과 계이름 연주 방법을 가장 직관적이고 사용자 친화적으로 배울 수 있는 웹앱”을 만드는 것이다.

앱의 핵심은 다음 5가지를 한 화면에서 연결하는 것이다.
1. 바이올린 현: G, D, A, E
2. 손가락 번호: 0번 개방현, 1번, 2번, 3번, 4번
3. 낮은/높은 손가락 위치: 낮은 1번, 보통 1번, 낮은 2번, 높은 2번, 3번, 높은 3번, 4번
4. 음이름: 예: G3, A4, C#5
5. 계이름: 고정도 기준 도, 레, 미, 파, 솔, 라, 시 및 #/♭ 표기

개발 원칙:
- 모든 음악 데이터는 한 곳에서 관리한다. UI에 음 데이터를 흩뿌려 하드코딩하지 않는다.
- 지판은 SVG 또는 HTML/CSS로 직접 그리며, 모바일에서도 손가락으로 누르기 쉬운 터치 영역을 만든다.
- 사용자가 지판의 한 위치를 누르면 해당 음이 강조되고, 정보 카드가 열리며, 소리가 재생된다.
- 기본 모드는 “1포지션 초급”이다.
- 초보자에게 혼란을 주지 않도록 화면의 기본 정보는 단순하게 유지하고, 고급 정보는 토글로 숨긴다.
- 계이름 모드는 기본적으로 고정도(C=도)를 사용하되, 추후 이동도 모드도 확장 가능하게 설계한다.
- 실제 바이올린은 프렛 악기가 아니므로 지판 위치는 “정확한 절대 위치”가 아니라 “이론적 위치와 청각 보정을 돕는 가이드”임을 안내한다.
- 모든 UI 텍스트는 한국어로 작성한다.
- 접근성을 고려해 색상만으로 의미를 전달하지 말고 텍스트, 아이콘, aria-label을 함께 사용한다.
- 외부 이미지나 음원에 의존하지 않고도 기본 기능이 동작해야 한다. 음원은 Web Audio API 또는 간단한 신시사이저 방식으로 생성할 수 있다.
- 코드는 TypeScript 기반으로 명확한 타입을 사용한다.

권장 기술 스택:
- React + TypeScript 기반 웹앱
- Next.js 또는 Vite 중 하나를 선택하되, 작은 교육용 앱이면 Vite도 가능하다.
- 스타일은 Tailwind CSS 또는 CSS Modules를 사용한다.
- 상태 관리는 처음에는 React state/context로 충분하게 구현하고, 필요해질 때만 가벼운 store를 추가한다.
- 오디오 재생은 Web Audio API를 기본으로 구현한다.
- 악보 표시가 필요하면 별도 컴포넌트로 분리하고, 초기 버전에서는 단순 오선/음표 SVG로 시작한다.

최종 산출물:
- 반응형 웹앱
- 지판 탐색 모드
- 계이름 연주 모드
- 단계별 레슨 모드
- 퀴즈 모드
- 진행률 저장 기능
- 초보자용 도움말
- 음악 데이터 테스트
- 배포 가능한 빌드

각 단계마다 다음을 반드시 포함하라.
1. 어떤 파일을 만들거나 수정할지
2. 핵심 컴포넌트 구조
3. 데이터 구조
4. 사용자가 실제로 보는 화면 동작
5. 완료 기준
6. 간단한 테스트 방법
```

## 2. 음악 데이터 기준

앱의 신뢰도는 음악 데이터의 정확도에서 시작합니다. 아래 구조를 “단일 진실 공급원”으로 삼으세요.

### 2.1 표준 조율

| 현 | 개방현 음 | MIDI | 고정도 계이름 | 설명 |
|---|---:|---:|---|---|
| G현 | G3 | 55 | 솔 | 가장 낮은 현 |
| D현 | D4 | 62 | 레 | 두 번째 낮은 현 |
| A현 | A4 | 69 | 라 | 표준 튜닝 기준음 |
| E현 | E5 | 76 | 미 | 가장 높은 현 |

### 2.2 1포지션 반음 운지 데이터

각 현에서 1포지션 기본 반음 위치는 다음처럼 계산합니다.

| 반음 오프셋 | 운지 라벨 | 초보자 설명 | 예: A현 기준 |
|---:|---|---|---|
| 0 | 0 | 개방현 | A4 |
| 1 | L1 | 낮은 1번 손가락 | A#4/B♭4 |
| 2 | 1 | 1번 손가락 | B4 |
| 3 | L2 | 낮은 2번 손가락 | C5 |
| 4 | H2 또는 2 | 높은 2번 손가락 | C#5/D♭5 |
| 5 | 3 | 3번 손가락 | D5 |
| 6 | H3 | 높은 3번 손가락 | D#5/E♭5 |
| 7 | 4 | 4번 손가락 | E5 |

> 교육용 기본 표시에서는 `0, 1, 2, 3, 4`를 우선 보여주고, `L1, L2, H2, H3`는 “반음 포함” 토글을 켰을 때 표시합니다.

### 2.3 네 현 전체 1포지션 표

| 현 | 0 | L1 | 1 | L2 | H2/2 | 3 | H3 | 4 |
|---|---|---|---|---|---|---|---|---|
| G현 | G3 | G#3/A♭3 | A3 | A#3/B♭3 | B3 | C4 | C#4/D♭4 | D4 |
| D현 | D4 | D#4/E♭4 | E4 | F4 | F#4/G♭4 | G4 | G#4/A♭4 | A4 |
| A현 | A4 | A#4/B♭4 | B4 | C5 | C#5/D♭5 | D5 | D#5/E♭5 | E5 |
| E현 | E5 | F5 | F#5/G♭5 | G5 | G#5/A♭5 | A5 | A#5/B♭5 | B5 |

### 2.4 고정도 계이름 매핑

| 음이름 | 계이름 |
|---|---|
| C | 도 |
| C# / D♭ | 도# / 레♭ |
| D | 레 |
| D# / E♭ | 레# / 미♭ |
| E | 미 |
| F | 파 |
| F# / G♭ | 파# / 솔♭ |
| G | 솔 |
| G# / A♭ | 솔# / 라♭ |
| A | 라 |
| A# / B♭ | 라# / 시♭ |
| B | 시 |

### 2.5 주파수 계산식

```ts
function midiToFrequency(midi: number): number {
  return 440 * Math.pow(2, (midi - 69) / 12);
}
```

### 2.6 지판 위치 계산식

4/4 바이올린 현장 기준을 옵션으로 둘 수 있습니다. 기본값은 `328.5mm`로 두되, UI에서는 mm 수치보다 상대 위치가 중요합니다.

```ts
function distanceFromNut(scaleLengthMm: number, semitoneOffset: number): number {
  return scaleLengthMm * (1 - Math.pow(2, -semitoneOffset / 12));
}
```

표시 위치는 실제 물리적 지판 길이에 맞춰 다음처럼 정규화합니다.

```ts
const relativePosition = distanceFromNut(scaleLengthMm, semitoneOffset) / distanceFromNut(scaleLengthMm, 7);
```

주의 문구:

```text
바이올린은 프렛이 없는 악기입니다. 표시된 위치는 초보자의 운지를 돕는 이론적 가이드이며, 실제 연주는 귀로 음정을 확인하며 미세 조정해야 합니다.
```

## 3. 앱 정보 구조

### 3.1 주요 화면

1. **홈**
   - 오늘의 학습 버튼
   - 지판 탐색 바로가기
   - 계이름 연주 바로가기
   - 퀴즈 바로가기
   - 진행률 요약

2. **지판 탐색**
   - 4개 현이 보이는 대형 인터랙티브 지판
   - 손가락 번호 표시 토글
   - 음이름 표시 토글
   - 계이름 표시 토글
   - 반음 위치 표시 토글
   - 왼손 보기 / 선생님이 보는 방향 전환
   - 탭하면 소리 재생 및 설명 카드 표시

3. **계이름 연주**
   - “도 레 미” 같은 계이름 시퀀스를 선택
   - 앱이 각 계이름을 어느 현과 몇 번 손가락으로 누를지 안내
   - 한 음씩 재생, 전체 재생, 따라 누르기 모드 제공

4. **레슨**
   - 개방현
   - 1번 손가락
   - 2번 손가락
   - 3번 손가락
   - 4번 손가락
   - D장조 음계
   - A장조 음계
   - G장조 음계
   - 반음 운지
   - 짧은 멜로디

5. **퀴즈**
   - “라를 A현에서 찾아보세요”
   - “D현 3번 손가락은 무슨 음일까요?”
   - “소리를 듣고 같은 위치를 눌러보세요”
   - “계이름을 보고 지판에서 순서대로 누르세요”

6. **도움말**
   - 현 이름 외우기
   - 손가락 번호 의미
   - 낮은/높은 손가락 개념
   - 계이름과 음이름 차이
   - 정확한 음정 잡는 법

7. **설정**
   - 계이름 방식: 고정도 / 이동도 확장 예정
   - 음 표기: # 우선 / ♭ 우선 / 조성 기반 자동
   - 왼손잡이 보기 또는 방향 반전
   - 소리 켜기/끄기
   - 메트로놈 BPM
   - 현장 길이 기준값

## 6. UI 컴포넌트 상세 요구사항

### 6.1 Fingerboard 컴포넌트

목표: 사용자가 가장 많이 보게 될 핵심 학습 화면입니다.

필수 기능:

- 4개 현을 수평 또는 수직으로 표시
- 각 현마다 0, 1, 2, 3, 4번 위치 표시
- “반음 보기”를 켜면 L1, L2, H3도 표시
- 각 위치는 최소 44px 이상의 터치 영역 확보
- 선택된 위치는 확대 또는 테두리 강조
- 선택 시 `SelectedNoteCard` 업데이트
- 선택 시 음 재생
- `aria-label` 예: `A현 1번 손가락 B4 시`

시각 설계:

- 현은 굵기가 살짝 다르게 표현: G가 가장 굵고 E가 가장 얇음
- 손가락 마커는 원형 버튼
- 마커 안에는 기본적으로 손가락 번호 표시
- 토글에 따라 `B4`, `시`, `1`을 함께 또는 따로 표시
- 초보 모드에서는 기본 5개 위치만 보여주고 반음은 숨김

정보 카드 예시:

```text
A현 1번 손가락
음이름: B4
계이름: 시
주파수: 493.88 Hz
연주 팁: A현에서 검지를 낮추지 말고 기본 1번 위치에 가볍게 올리세요.
```

### 6.2 SolfegePlay 컴포넌트

목표: 계이름을 보고 어떤 현과 손가락으로 연주할지 바로 알 수 있게 합니다.

필수 기능:

- 계이름 시퀀스 선택: 예: `레 미 파# 솔 라`, `라 시 도# 레 미`
- 각 계이름을 지판 위치와 연결
- 현재 연주해야 할 음을 크게 표시
- “한 음 듣기”, “전체 듣기”, “따라 누르기” 버튼
- 사용자가 올바른 위치를 누르면 다음 음으로 이동
- 틀리면 정답 위치를 은은하게 힌트 표시

예시 화면 문구:

```text
오늘의 계이름: 레 - 미 - 파# - 솔 - 라
시작 현: D현
운지: 0 - 1 - 2 - 3 - 4
```

### 6.3 LessonPlayer 컴포넌트

목표: 사용자가 “무엇부터 배워야 하는지” 고민하지 않게 합니다.

필수 기능:

- 한 레슨은 3~7개의 짧은 step으로 구성
- 각 step은 한 가지 행동만 요구
- 예: “D현 개방현을 눌러보세요”, “D현 1번 손가락 E를 찾아보세요”
- 단계 완료 시 즉시 피드백
- 레슨 종료 시 별점 또는 배지 제공

### 6.4 QuizEngine 컴포넌트

목표: 사용자가 배운 내용을 게임처럼 확인합니다.

퀴즈 유형:

1. 음이름 → 위치 찾기
2. 계이름 → 위치 찾기
3. 위치 → 음이름 맞히기
4. 위치 → 계이름 맞히기
5. 소리 듣고 위치 찾기
6. 짧은 계이름 시퀀스 따라 누르기

피드백 원칙:

- 정답: “좋아요! A현 1번 손가락은 B4, 계이름은 시입니다.”
- 오답: “거의 맞았어요. 지금 누른 곳은 C5이고, 정답은 B4입니다.”
- 초보자에게 `틀렸습니다`보다 `다시 한 번 찾아볼까요?`를 사용

## 8. 레슨별 상세 콘텐츠 설계

### 레슨 1: 네 개의 개방현 익히기

목표:

- G, D, A, E 현 이름을 알고 소리를 구분한다.
- 손가락을 누르지 않는 `0번`의 의미를 이해한다.

단계:

1. G현 듣기: “가장 낮은 솔 소리입니다.”
2. D현 듣기: “레 소리입니다.”
3. A현 듣기: “라 소리입니다. 튜닝 기준으로 자주 사용합니다.”
4. E현 듣기: “가장 높은 미 소리입니다.”
5. 퀴즈: “라 소리가 나는 개방현을 눌러보세요.”

완료 조건:

- 4개 개방현을 각각 1번 이상 올바르게 누르기
- 개방현 퀴즈 3문제 중 2문제 이상 정답

### 레슨 2: 1번 손가락 익히기

목표:

- 각 현에서 1번 손가락의 기본 위치를 익힌다.

핵심 음:

| 현 | 1번 손가락 | 계이름 |
|---|---|---|
| G현 | A3 | 라 |
| D현 | E4 | 미 |
| A현 | B4 | 시 |
| E현 | F#5 | 파# |

단계:

1. G현 1번 A3 찾기
2. D현 1번 E4 찾기
3. A현 1번 B4 찾기
4. E현 1번 F#5 찾기
5. “같은 1번 손가락이라도 현이 바뀌면 음이 달라진다” 안내

| 현 | 2번 손가락 | 계이름 |

### 레슨 5: 4번 손가락과 다음 개방현

목표:

핵심 관계:

| 낮은 현 4번 | 같은 음 | 높은 현 개방현 |
|---|---|---|
| G현 4번 | D4 | D현 0번 |
| D현 4번 | A4 | A현 0번 |
| A현 4번 | E5 | E현 0번 |
| E현 4번 | B5 | 다음 현 없음 |

학습 UX:

- 두 위치를 동시에 하이라이트한다.
- 먼저 낮은 현 4번을 듣고, 바로 다음 개방현을 들려준다.

## 11. 악보·계이름 확장 설계

초기 버전에서는 지판 중심으로 만들고, 이후 다음 기능을 확장하세요.

### 11.1 간단 오선 표시

- 선택한 음을 작은 오선 위에 표시
- 처음에는 정교한 악보 엔진 없이 SVG로 단순 구현
- 높은음자리표 기준으로 D4~B5 정도만 표시해도 초급에는 충분
- 각 음표 아래에 계이름 표시

### 11.2 이동도 계이름 확장

기본은 고정도이지만, 추후 이동도 모드를 추가할 수 있습니다.

필요 설정:

- 조성 선택: C, G, D, A, F, B♭
- 장조/단조 선택
- 선택한 조성에서 1도=도 방식으로 계이름 계산

주의:

- 초보자에게는 고정도와 이동도를 동시에 보여주면 혼란스러울 수 있으므로 설정에서 명확히 분리합니다.

### 11.3 실제 연주 입력 확장

추후 마이크 입력을 통해 사용자가 실제 바이올린으로 낸 음을 분석할 수 있습니다.

초기에는 구현하지 말고, 구조만 확장 가능하게 둡니다.

- getUserMedia 기반 마이크 입력
- pitch detection
- 목표 음과 실제 음 비교
- 너무 높음 / 너무 낮음 피드백

## 19. 개발자가 특히 주의해야 할 점

1. **계이름과 음이름을 혼동하지 않기**  
   한국어 학습자에게 `도레미`는 계이름처럼 느껴지지만, 앱 기본은 고정도입니다. 설정과 도움말에서 명확히 설명하세요.

2. **프렛처럼 보이게 만들지 않기**  
   지판 위치는 시각 가이드입니다. “정확한 위치”라고 표현하지 말고 “기준 위치”, “가이드 위치”라고 표현하세요.

3. **반음을 처음부터 많이 보여주지 않기**  
   초보자는 L1, L2, H3가 한꺼번에 보이면 혼란스럽습니다. 기본 화면에서는 숨기고, 토글로 보여주세요.

4. **음악 데이터를 UI에 직접 쓰지 않기**  
   `A현 1번 = B4` 같은 정보를 여러 컴포넌트에 직접 적으면 오류가 생깁니다. 반드시 데이터 함수에서 가져오세요.

5. **소리 없는 환경도 고려하기**  
   학교, 지하철, 모바일 브라우저 등에서 소리가 막힐 수 있습니다. 소리 없이도 정보 카드와 지판 학습은 가능해야 합니다.

6. **정답/오답 피드백을 친절하게 쓰기**  
   초보 학습자는 실수에 민감합니다. “틀림”보다 “거의 맞았어요. 한 칸만 더 가까운 위치예요.”처럼 안내하세요.

7. **모바일에서 먼저 테스트하기**  
   이 앱은 손가락으로 지판을 누르는 경험이 핵심입니다. 데스크톱보다 모바일 터치 경험을 먼저 확인하세요.

## 20. 완성 앱의 이상적인 첫 사용 흐름

1. 사용자가 홈에 들어온다.
2. “지판 탐색 시작”을 누른다.
3. A현 개방현을 누른다.
4. 앱이 `A현 0번 / A4 / 라 / 소리`를 보여준다.
5. A현 1번을 누른다.
6. 앱이 `B4 / 시 / 검지로 누르기`를 보여준다.
7. 사용자가 “계이름 연주”로 이동한다.
8. `라 시 도# 레 미`를 선택한다.
9. 앱이 `A현 0-1-2-3, E현 0`을 순서대로 안내한다.
10. 사용자가 지판을 따라 누르며 짧은 멜로디를 완성한다.
11. 앱이 “좋아요! 이제 A현과 E현의 기본 손가락을 연결했어요.”라고 피드백한다.
12. 사용자는 퀴즈로 넘어가 자신이 배운 내용을 확인한다.

이 설계서의 핵심은 “음악 이론 설명 앱”이 아니라, **손가락으로 눌러보며 즉시 이해하는 바이올린 학습 도구**를 만드는 것입니다. 모든 개발 결정은 초보자가 `어디를 누르고, 어떤 소리가 나며, 그 계이름이 무엇인지`를 더 빨리 이해하도록 돕는 방향으로 판단하세요.
