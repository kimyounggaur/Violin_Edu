# 🎻 바이올린 운지법 마스터 — 모바일 웹앱 바이브코딩 프롬프트 (통합 결정판)

> 첨부된 세 프롬프트(ChatGPT·Claude·Gemini)의 장점만 결합한 **모바일 우선(Mobile-First)** 설계서입니다.
> - **Gemini**에서 → 강한 역할 설정 · Notion풍 미니멀 디자인 · Framer Motion 모션 · 정답/오답 애니메이션
> - **ChatGPT**에서 → 단일 데이터 소스 원칙 · MIDI/주파수/지판 물리 위치 공식 · "프렛 없음" 교육 정확성 · 타입 설계 · 친절한 문구 · 개발 보조 프롬프트
> - **Claude**에서 → 복붙형 단계별 STEP 프롬프트 · 검증된 운지 데이터 · 손가락 색상 체계 · "2·3번이 붙는 반음" 시각화 · 연습곡 데이터
>
> **핵심 차별점**: 손가락으로 지판을 눌러보는 *모바일 터치 경험*이 이 앱의 전부입니다. 모든 결정은 "한 손으로, 한 번 탭해서, 즉시 이해"를 향합니다.

## 0. 이 문서 사용법

1. 어떤 AI 코딩 도구든(Claude, Cursor, Bolt.new, Lovable, v0, Replit Agent 등) 사용 가능합니다.
2. **[부록 A. 마스터 프롬프트]를 먼저 붙여넣어** 전체 맥락을 심습니다.
3. 그다음 **[6. 단계별 빌드 프롬프트] STEP 0 → STEP 9**를 순서대로 붙여넣고, 각 단계의 **확인 포인트**를 통과한 뒤 다음으로 넘어갑니다.
4. 한 번에 만들고 싶으면 [부록 A]만 써도 되지만, 단계별 진행이 품질이 훨씬 좋습니다.

> ⚠️ **가장 중요한 규칙**: [4. 음악 데이터]는 음악적으로 검증된 값입니다. **AI가 음높이·손가락 위치를 임의로 추측하지 못하게** 하고, 반드시 "단일 데이터 소스에서 계산"하도록 명시하세요.

## 1. 역할 & 제품 정의

```text
[역할] 너는 음악교육 UX에 강한 시니어 프론트엔드 엔지니어이자, 모바일 앱 UI/UX 디자이너이며,
바이올린 초급 교육 콘텐츠 설계자다. 너는 "모바일에서 손가락으로 만지는 학습 경험"을 최우선으로 설계한다.
```

- **제품명(예시)**: `Fiddle Tap — 바이올린 운지법 마스터` (자유롭게 변경 가능)
- **한 문장 정의**: *바이올린 지판을 탭하면 현·손가락 번호·음이름·계이름·소리·악보 위치가 즉시 연결되고, 짧은 미션으로 운지법과 계이름 연주를 자연스럽게 익히는 모바일 웹앱.*
- **핵심 사용자**: 바이올린을 처음 배우는 어린이·청소년·성인, "악보의 도레미는 알지만 바이올린 어디를 누를지 모르는" 학습자, 수업 보조 교구가 필요한 교사.
- **최우선 UX 원칙**
  1. **한 번 탭하면 바로 이해**된다. (탭 → 강조 + 정보 + 소리 + 진동이 0.1초 내 동시 반응)
  2. 화면에는 항상 `현 이름 · 손가락 번호 · 음이름 · 계이름 · 소리 듣기`가 함께 보인다.
  3. 초보자가 혼란스럽지 않게 **기본 정보는 단순**, 고급(반음 등)은 **토글로 숨김**.
  4. 바이올린은 프렛이 없으므로 모든 위치는 "정확한 절대 위치"가 아니라 **귀로 보정하는 가이드**임을 안내한다.
  5. **모바일에서 먼저 완성**한다. 데스크톱은 그 다음.

## 3. 📱 모바일 UI/UX 설계 (이 문서의 핵심)

### 3.1 화면 레이아웃 (세로/포트레이트 우선)

지판이 화면의 주인공이고, 위에는 "지금 만진 음" 피드백, 아래에는 엄지로 닿는 탭바를 둡니다.

```
┌───────────────────────────────┐ ← safe-area-inset-top
│ 🎻 운지 마스터          ⚙︎  🔊 │  ① 앱바 (≈56px, 고정)
├───────────────────────────────┤
│  ┌──────── 선택한 음 ────────┐ │
│  │   시        B4        ▶  │ │  ② 선택음 디스플레이 (sticky)
│  │   A현 · 1번 손가락(검지)   │ │     - 큰 계이름 + 음이름 + 현/손가락
│  │   ♪ ─●───── (미니 오선보)  │ │     - ▶ 다시 듣기
│  └───────────────────────────┘ │
├───────────────────────────────┤
│   G     D     A     E          │ ← 줄 라벨(항상 보임)
│   ●     ●     ●     ●    0     │
│                                │
│   ⬤     ⬤     ⬤     ⬤    1 검지 │  ③ 지판(히어로) — 남는 공간 전부
│   ⬤     ⬤     ⬤     ⬤    2 중지 │     · physics 위치로 배치
│   ⬤     ⬤     ⬤     ⬤    3 약지 │     · 2·3번은 서로 붙음(반음)
│                                │
│   ⬤     ⬤     ⬤     ⬤    4 소지 │
│                                │
│  ╴╴╴╴╴ [ 보기 옵션 ▲ ] ╴╴╴╴╴   │ ← 바텀시트 핸들(토글 모음)
├───────────────────────────────┤
│ [탐색]  [계이름]  [퀴즈]  [연습곡]│  ④ 하단 탭바 (thumb zone, ≈64px)
└───────────────────────────────┘ ← safe-area-inset-bottom
```

- **③ 지판이 화면의 55% 이상**을 차지하는 히어로. 위가 너트(개방현), 아래로 갈수록 음이 높아짐.
- **보기 옵션(계이름/음이름/손가락 토글, 반음 토글 등)은 바텀시트**로 — 손이 화면 상단까지 가지 않게.
- **데스크톱/가로**에서는 좌(지판)–우(디스플레이) 분할로 자동 전환.

### 3.2 디자인 시스템 (Notion풍 미니멀 + 악기의 우드 감성)

깨끗하고 군더더기 없는 Notion 스타일을 기본으로 하되, 지판만 흑단(ebony) 질감으로 시선을 집중시킵니다.

| 토큰 | 라이트 | 다크(OLED 친화) |
|---|---|---|
| 배경 | `#F9F9F7` | `#0E0E10` |
| 표면/카드 | `#FFFFFF` | `#1B1B1F` |
| 지판(흑단) | `#241E18` | `#15110D` |
| 텍스트 | `#2D2D2D` | `#F3F3F3` |
| 보조 텍스트 | `#8A8275` | `#9A958C` |
| 주 색상(액션) | `#2EAADC` (Notion 블루) | `#5BC2EC` |
| 정답 | `#16A34A` · 오답 `#DC2626` | 동일 |

**손가락 색상(앱 전체 일관 · 색만으로 구분 금지, 항상 번호 동반)**

| 손가락 | 색상 | 손 |
|---|---|---|
| 0 개방현 | `#64748B` (슬레이트, 빈 원) | 누르지 않음 |
| 1번 | `#2563EB` (파랑) | 검지 |
| 2번 | `#16A34A` (초록) | 중지 |

- **폰트**: `Pretendard` → fallback `system-ui, -apple-system, sans-serif`. 숫자/음이름은 또렷하게, 제목은 굵게.
- **모서리/그림자**: 카드 `rounded-2xl`, 버튼 `rounded-xl`, 미세 그림자(`shadow-sm`)만. 장식 최소화.
- **타입 스케일(모바일)**: 선택음 계이름 32–40px, 음이름 18px, 본문 15px, 캡션 12px.

### 3.3 터치 & 제스처 (모바일 필수)

- **모든 탭 타깃 ≥ 48×48px** (음 마커는 패딩 넉넉히, 시각 원이 작아도 히트영역은 크게).
- **엄지 존(thumb zone)**: 주요 조작(탭바·보기옵션·다시듣기)은 화면 하단 절반에 배치.
- **바텀시트**: 보기 옵션·곡 선택 등은 아래에서 슬라이드 업. 핸들 드래그/탭으로 열고 닫기.
- **스와이프**: 탭바 모드 좌우 스와이프로도 전환(선택).
- **safe-area**: `env(safe-area-inset-*)`로 노치/홈 인디케이터 침범 방지.
- **확대/선택 방지**: `touch-action: manipulation`(더블탭 확대 막기), 컨트롤에 `user-select:none`, `-webkit-touch-callout:none`, `-webkit-tap-highlight-color: transparent`.
- **뷰포트 메타**: `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">`.

### 3.4 햅틱 & 사운드 (즉각 피드백)

- **햅틱**: 지원 시 `navigator.vibrate()` — 탭 `10ms`, 정답 `[10,40,10]`, 오답 `[60]`. (미지원 기기에서 무시되게 feature-check)
- **오디오 언락**: 모바일은 첫 사용자 제스처에서만 소리가 켜짐 → 첫 탭에서 `await Tone.start()` 호출.
- **저지연**: 시각·청각·햅틱이 **동시에**(50ms 이내) 반응. 음색은 부드러운 신스(`Tone.AMSynth`/`FMSynth` + 적당한 attack/release)로 바이올린 느낌 근사.
- **무음 환경 대응**: 소리가 막혀도 정보 카드·지판 학습은 정상 동작(소리 켜기/끄기 토글 제공).

### 3.5 모션 (Framer Motion 사양)

- **탭**: 마커 `scale: 1 → 1.12 → 1` 스프링(`stiffness 400, damping 20`), 활성 색으로 채움.
- **선택 강조**: 활성 마커에 은은한 글로우 링(`layoutId`로 부드럽게 이동).
- **정답**: 초록 링이 퍼지며 사라짐(`scale 0→1.8`, `opacity 1→0`, 400ms).
- **오답**: 가벼운 좌우 흔들림(`x: [0,-8,8,-6,6,0]`, 300ms) + 빨강 테두리 깜빡.
- **모드 전환**: 슬라이드+페이드(150–250ms).
- **접근성**: `prefers-reduced-motion` 존중 → 애니메이션 최소화/즉시 전환. 과한 효과는 금지.

### 3.6 PWA (설치형 + 오프라인)

- `manifest.json`: `name`, `short_name`, 아이콘(192/512), `theme_color`, `background_color`, `display: "standalone"`, `orientation: "portrait"`.
- **서비스워커**로 앱 셸·정적 자원 캐싱 → 비행기 모드에서도 지판 학습 가능.
- 설치 안내(Add to Home Screen) 배너는 은은하게 1회만.

## 4. 🎼 음악 데이터 (단일 진실 공급원 — 반드시 그대로 사용)

> 앱의 신뢰도는 데이터 정확도에서 시작합니다. **모든 음 정보는 아래 데이터에서 "계산"으로 도출**하고, UI 컴포넌트에 음 정보를 직접 하드코딩하지 마세요.

### 4.1 표준 조율 · MIDI · 주파수

| 현 | 개방현 | MIDI | 주파수(Hz) | 계이름 |
|---|---|---:|---:|---|
| G현 (4번줄) | G3 | 55 | 196.00 | 솔 |
| D현 (3번줄) | D4 | 62 | 293.66 | 레 |
| A현 (2번줄) | A4 | 69 | 440.00 | 라 |
| E현 (1번줄) | E5 | 76 | 659.25 | 미 |

### 4.2 제1포지션 기본 운지표 (기본 5위치 — 초보 모드 기본값)

**높은 2번(High-2) 형태**로 통일 — D장조/A장조 계열에서 자연스러운 표준 배열입니다.

| 현 | 0 개방현 | 1번(검지) | 2번(중지) | 3번(약지) | 4번(소지) |
|---|---|---|---|---|---|
| **G현** | G3 · 솔 | A3 · 라 | B3 · 시 | C4 · 도 | D4 · 레 |
| **D현** | D4 · 레 | E4 · 미 | F♯4 · 파♯ | G4 · 솔 | A4 · 라 |
| **A현** | A4 · 라 | B4 · 시 | C♯5 · 도♯ | D5 · 레 | E5 · 미 |
| **E현** | E5 · 미 | F♯5 · 파♯ | G♯5 · 솔♯ | A5 · 라 | B5 · 시 |

**손가락 간격(핵심 시각 포인트) — 모든 줄 공통**

```
개방현(0) —온음— 1번 —온음— 2번 —반음— 3번 —온음— 4번
                            └─ 2·3번 손가락이 서로 붙음(반음) ─┘
```

- 💡 **앱에 표시할 꿀팁**: 각 줄의 **4번 손가락 음 = 바로 윗줄의 개방현 음** (예: G현 4번 = D = D현 개방현). 인토네이션 확인용.
- ⚠️ **데이터 정합성 주의**: E현 2번은 **G♯(솔♯)** 입니다(High-2 통일 기준). 일부 교본의 `E현 2번 = G(솔, 낮은 2번)`은 **§4.3 반음(낮은 손가락) 토글**에서 다룹니다 — 기본 화면에 섞지 마세요.

### 4.3 반음 포함 확장 (토글로만 노출)

초보 기본 화면에는 0·1·2·3·4만 보이고, **"반음 포함" 토글**을 켜면 낮은/높은 손가락을 추가로 표시합니다.

| 반음 오프셋 | 라벨 | 초보 설명 | 예: A현 |
|---:|---|---|---|
| 0 | 0 | 개방현 | A4 |
| 1 | L1 | 낮은 1번 | A♯4 / B♭4 |
| 2 | 1 | 1번 | B4 |
| 3 | L2 | 낮은 2번 | C5 |
| 4 | 2 (High-2) | (기본) 2번 | C♯5 / D♭5 |
| 5 | 3 | 3번 | D5 |
| 6 | H3 | 높은 3번 | D♯5 / E♭5 |
| 7 | 4 | 4번 | E5 |

### 4.4 지판 위치 계산식 (프렛 없는 악기의 정확한 배치)

마커를 균등 간격이 아니라 **실제 비율**로 배치합니다(아래로 갈수록 간격이 좁아지고, 2·3번이 자연히 가까워짐).

```ts
// 어떤 반음 오프셋이든 제1포지션 내 상대 위치(0=너트, 1=4번손가락)로 변환
const SCALE = 328;            // 4/4 바이올린 현장(mm) 기준
const dist = (semi: number) => SCALE * (1 - Math.pow(2, -semi / 12));
export const posRatio = (semi: number) => dist(semi) / dist(7);
```

```ts
export type StringId = "G" | "D" | "A" | "E";

export const PITCH_SHARP = ["C","C#","D","D#","E","F","F#","G","G#","A","A#","B"] as const;
export const SOLFEGE_SHARP = ["도","도#","레","레#","미","파","파#","솔","솔#","라","라#","시"] as const;

export const STRINGS = [
  { id: "G", koreanName: "G현", strNo: 4, openMidi: 55, order: 0 },
  { id: "D", koreanName: "D현", strNo: 3, openMidi: 62, order: 1 },
  { id: "A", koreanName: "A현", strNo: 2, openMidi: 69, order: 2 },
  { id: "E", koreanName: "E현", strNo: 1, openMidi: 76, order: 3 },
] as const;

// 기본 5위치(초보) + 반음 확장. isChromatic=true 는 "반음 포함" 토글에서만 노출
export const FINGERS = [
  { label: "0",  finger: 0, semi: 0, isChromatic: false, name: "개방현", handName: "" },
  { label: "L1", finger: 1, semi: 1, isChromatic: true,  name: "낮은 1번", handName: "검지" },
  { label: "1",  finger: 1, semi: 2, isChromatic: false, name: "1번",     handName: "검지" },
  { label: "L2", finger: 2, semi: 3, isChromatic: true,  name: "낮은 2번", handName: "중지" },
  { label: "2",  finger: 2, semi: 4, isChromatic: false, name: "2번",     handName: "중지" },
  { label: "3",  finger: 3, semi: 5, isChromatic: false, name: "3번",     handName: "약지" },
  { label: "H3", finger: 3, semi: 6, isChromatic: true,  name: "높은 3번", handName: "약지" },
  { label: "4",  finger: 4, semi: 7, isChromatic: false, name: "4번",     handName: "소지" },
] as const;

export const FINGER_COLORS: Record<number,string> =
  { 0:"#64748B", 1:"#2563EB", 2:"#16A34A", 3:"#EA580C", 4:"#9333EA" };

// ── 계산 함수 (여기서만 음 정보가 만들어진다) ──
export const midiToName    = (m:number) => `${PITCH_SHARP[m%12]}${Math.floor(m/12)-1}`; // 60→C4
export const midiToSolfege = (m:number) => SOLFEGE_SHARP[m%12];
export const midiToFreq    = (m:number) => 440 * Math.pow(2, (m-69)/12);

const SCALE = 328;
const dist = (s:number) => SCALE * (1 - Math.pow(2, -s/12));
export const posRatio = (s:number) => dist(s) / dist(7); // 0(너트)~1(4번)

// 한 위치의 모든 표시 정보를 생성
export function getNote(stringId: StringId, fingerLabel: string) {
  const str = STRINGS.find(s => s.id === stringId)!;
  const f   = FINGERS.find(x => x.label === fingerLabel)!;
  const midi = str.openMidi + f.semi;
  return {
    stringId, fingerLabel: f.label, finger: f.finger,
    note: midiToName(midi),          // 예: "F#4"
    solfege: midiToSolfege(midi),    // 예: "파#"
    midi, freq: midiToFreq(midi),
    posRatio: posRatio(f.semi),      // 지판 세로 위치(0~1)
    color: FINGER_COLORS[f.finger],
    isChromatic: f.isChromatic,
    aria: `${str.koreanName} ${f.label === "0" ? "개방현" : f.label + "번 손가락"}, ${midiToSolfege(midi)} ${midiToName(midi)}`,
  };
}
```
